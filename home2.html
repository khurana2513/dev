Load Classification

Lights, Fans & Sockets
Are NOT the Same Load

Each load type has different electrical behaviour. Treating them the same is the #1 amateur mistake — it causes relay failure, fire hazards, and poor control.

Your Three Load Types — Key Differences

💡 Light (Resistive)
0.5A – 1A per bulb

LED: purely resistive. No inrush. Simplest load. Any relay handles this easily. Standard relay wiring, NO port, done.

🌀 Ceiling Fan (Inductive)
0.3A – 0.5A running, 3–5× inrush

Inductive motor. Creates back-EMF on switch-off. Needs a snubber circuit or rated relay. Speed needs a TRIAC/dimmer, NOT relay.

🔌 Socket (Unknown)
Up to 6A / 16A depending on rating

Unknown device will be plugged in. Could be a phone charger or an iron. Must use a high-rated relay AND add protection. Consider carefully.

⚠ Critical — Fan Inrush Current

A ceiling fan draws 3–5× its running current at startup (inrush). A fan rated at 0.4A running may pull 1.5–2A for the first 100ms. This causes relay contact arcing over time. Always use a relay rated at ≥10A for fans, and add a snubber (0.1µF + 100Ω in series across the relay contacts).

RELAY MODULE RECOMMENDATION

8-Ch Opto-Isolated Module

SRD-05VDC relays, opto-isolated inputs, 10A contacts. Handles lights perfectly. Use for all light circuits. ~₹350-500.

RobotDyn AC Dimmer

TRIAC-based, 1–8 channel, 3.3V logic compatible. Zero-cross detection built-in. Required for fan speed control. ~₹250 per channel.

16A Relay (per socket)

HJQ-78F or similar rated 16A/250V. One per socket you want to control. Don't use 10A relays on sockets — too risky. ~₹80-120 each.

PCF8574 I²C Expander

Adds 8 GPIO pins over just 2 wires (SDA/SCL). Essential since NodeMCU only has ~11 usable GPIO and you need many inputs + outputs. ~₹80.

Per-Device Wiring (Parallel Method — The Only Right Way)

  ─── FOR LIGHTS (Simple) ──────────────────────────────────────────

  LIVE ─────┬──── [PUSH BUTTON] ──────────────┬──── [LED BULB] ──── NEUTRAL
            └──── [RELAY COM → NO] ────────────┘

  NodeMCU GPIO (Push Btn Input, INPUT_PULLUP) ←─── Push Button ──── GND
  NodeMCU GPIO (Relay Output, active LOW)     ──→  Relay IN pin


  ─── FOR FANS (ON/OFF + Speed) ────────────────────────────────────

  LIVE ─────────────── [RELAY COM → NO] ──── [AC DIMMER IN] ──── [FAN] ──── NEUTRAL
                                                     ▲
  NodeMCU UART TX  ──────────────────────────────────┘ (PWM signal for speed)
  NodeMCU GPIO     ──→  Relay IN pin (master ON/OFF)

  IMPORTANT: Relay controls master power. Dimmer controls speed.
  Set dimmer to last-known level before energizing relay to avoid inrush spike.


  ─── FOR SOCKETS ───────────────────────────────────────────────────

  LIVE ─── [6A Fuse] ─── [16A RELAY COM → NO] ─── [SOCKET LIVE TERMINAL]
  NEUTRAL ─────────────────────────────────────── [SOCKET NEUTRAL TERMINAL]
  EARTH ───────────────────────────────────────── [SOCKET EARTH TERMINAL]

  NodeMCU GPIO ──→ 16A Relay IN pin

  ALWAYS fuse the socket circuit. If a short circuit occurs in the plugged
  device, the fuse blows — not your relay or wiring.
✓ Why Push Buttons, Not Toggle Switches

Push buttons feed into NodeMCU GPIO and trigger a TOGGLE in software. This means relay state is always tracked in software — no mismatch between physical switch position and relay state. The relay is the ONLY thing that controls device power. This is exactly how Sonoff, Shelly, and all commercial smart switches work. Replace your existing rockers with momentary push buttons (₹15–30 each).

☠ Boot Pin Rules — Read This

GPIO0 and GPIO2 must be HIGH at boot, GPIO15 must be LOW at boot — these are ROM boot mode pins. If you have active-LOW relay modules connected to GPIO0/2 and the module pulls them LOW, the NodeMCU enters flash mode and won't run. Fix: use opto-isolated relay modules (the isolator keeps GPIO state stable), initialize these pins HIGH in the very first line of setup(), or use GPIO expanders for these relay channels.

Fan Speed Control

Variable Fan Speed
via TRIAC Dimmer

A relay is binary. Fan speed control requires phase-cutting the AC waveform — that's a TRIAC, not a relay.

How AC Fan Speed Control Works

  Normal AC sine wave:          ─╱╲──╱╲──╱╲─  (full voltage = full speed)

  Phase-cut (50% power):        ──╱╲────╱╲───  (half wave = ~70% speed)

  Phase-cut (25% power):        ───╲──────╲──  (quarter wave = ~40% speed)

  The TRIAC turns ON partway through each half-cycle.
  Zero-cross detector tells the MCU exactly when the sine crosses 0V,
  so it can time the TRIAC trigger precisely for consistent dimming.

  NodeMCU → [RobotDyn Dimmer] → Fan motor
  PWM value 0–255 → maps to 0–100% phase angle

Capacitor-Based Speed Control (Traditional, No MCU)

  Traditional fan regulators use series capacitors to reduce voltage.
  Avoid this method for IoT — you can't control capacitor switching digitally
  without complex relay banks. Use the TRIAC dimmer instead.

  However: keep the existing capacitor (2µF–4µF) in the fan itself for
  starting torque — it's part of the fan motor circuit, not the regulator.
⚠ Compatibility Warning

TRIAC dimmers work with inductive motor fans (most ceiling fans). They do NOT work with BLDC fans (inverter fans). Check your fan type. If your fan has a remote or inverter label, use relay ON/OFF only and control speed via the fan's own IR remote using an IR blaster on NodeMCU instead.

Socket Automation

Smart Sockets —
Do It Right or Don't

Automating a wall socket is the highest-risk part of this project. The device you'll plug in is unknown at installation time.

Decision Tree — Should You Automate This Socket?

  What will be plugged into this socket?
           │
           ├── Phone charger / laptop / LED strip / small appliances (≤1000W)
           │          → YES, automate with a 16A relay + 6A fuse
           │
           ├── Electric iron / hair dryer / room heater (1000–2000W)
           │          → MAYBE — use a 16A relay, 16A fuse, thick wire (2.5mm²)
           │            Consider: is the socket always used for this device?
           │
           ├── Air conditioner / water heater / washing machine (>2000W)
           │          → NO — use a smart plug/contactor designed for the load
           │            Never control these from a standard relay module
           │
           └── Unknown / general use socket
                      → Automate ONLY with proper protection:
                        16A relay + 10A fuse + RCBO on the circuit
✓ Professional Approach for General Sockets

If the socket is for general use, use a smart plug (like TP-Link Tapo P100 or Mi Smart Plug) in the socket instead of wiring a relay behind it. Flash Tapo with custom firmware, or integrate via the Tapo integration in Home Assistant. This way the socket is still physically accessible and you haven't wired high-current relays behind a panel that might handle unknown loads.

Setup Path — Google Home

  1. Set up Home Assistant on RPi/PC with your ESPHome devices auto-discovered
  2. Subscribe to Nabu Casa (~₹500/month) — this gives HA a public HTTPS endpoint without port forwarding
  3. In HA Settings → Voice Assistants → enable Google Assistant
  4. Link in the Google Home app → Add Device → Works with Google → Home Assistant
  5. All your HA entities (switches, fan) appear in Google Home automatically
  6. Say "Hey Google, turn on Board1 Light" — or rename entities to natural names like "room light", "ceiling fan"

Setup Path — Amazon Alexa

  1. Same steps 1–2 as above (HA + Nabu Casa)
  2. In HA Settings → Voice Assistants → enable Amazon Alexa
  3. In the Alexa app → Skills → search Home Assistant → Enable → sign in with Nabu Casa
  4. Discover devices in Alexa — all HA entities appear
  5. Rename entities to friendly names in HA for natural voice commands
ℹ Free Alternative (No Nabu Casa)

Use Cloudflare Tunnel (free) to expose Home Assistant publicly. Then use the manual Alexa skill or Google Assistant manual setup in HA. More complex to configure but zero monthly cost. Or stay fully local — use the Home Assistant app which tunnels via Nabu Casa or Cloudflare, and use on-device Google Assistant via HA's local voice assistant feature (requires a mic-enabled device on the local network).

Safety

Non-Negotiable
Safety Rules

You're working with 230V AC mains. Every rule below exists because someone was hurt ignoring it.

☠ ALWAYS isolate mains before working

Turn off the MCB for the circuit you're working on. Verify with a non-contact voltage tester before touching any wire. 230V AC at typical home impedance is fatal. Do not work on live circuits under any circumstances.
